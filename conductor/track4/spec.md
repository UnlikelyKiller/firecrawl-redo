# Specification: Track 4 - ModelAdapter + Structured Extraction

## Overview

Track 4 gives CrawlX a capability-aware extraction layer that converts page content and receipts into validated structured outputs. In the updated plan, this track must also remain compatible with richer page-read surfaces coming from Tandem or other external backends.

## Deliverables

- adapter and router plan
- extraction pipeline plan
- provider compatibility plan
- extraction test plan

## Acceptance Criteria

- extraction remains backend-agnostic
- schema validation and repair flow are explicit
- prompt and source handling rules are documented

## Dependencies

- Track 1
- Track 3
