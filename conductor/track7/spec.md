# Specification: Track 7 - Dashboard + Activity + Operations Visibility

## Overview

Track 7 plans the operator surface for CrawlX. In the updated plan, it must do more than show jobs and receipts. It must expose enough information to operate external browser sessions safely:

- which profile was used
- whether a lease is active
- whether a profile is quarantined
- what backend was attached

## Deliverables

- dashboard page plan
- external backend visibility plan
- artifact rendering plan
- operator-debugging support plan

## Acceptance Criteria

- operators can understand job/backend/profile state from the UI
- receipt rendering behavior is explicit
- API/UI contracts are planned and testable
